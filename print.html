Getting Started

Installation

Currently v1.0.0-alpha.6

Prerequisites

  • Install Rust
  • Setup Rust:
rustup override set stable && rustup update

Clone Cairo 1.0 compiler

Clone the repository locally

git clone https://github.com/starkware-libs/cairo

Build the repository

cd cairo
cargo build --all --release
cd target/release
run pwd # to know the path to the binaries (e.g. starknet-compile)

Add the release directory to PATH

  • Open ~/.zshrc or ~/.bashrc (depending your shell)
  • Add the following line to the end of the file:
export PATH="<PATH_TO_CAIRO_COMPILER>"

e.g.

export PATH="/Users/username/cairo/target/release"
  • Save the change and restart terminal.
  • To check run
which starknet-compile

Should see the path to the Cairo 1.0 repo printed!, /Users/username/cairo/target/release/cairo-compile

Hello, World!

Creating a Project Directory

Writing and Running a Cairo Program

mkdir ~/projects
cd ~/projects
mkdir hello_world
cd hello_world
  • Create new file main.cairo and add the following code:
fn main() {
    debug::print_felt252('Hello, world!')
}
  • Save the file and go back to your terminal window in the ~/projects/hello_world directory. On Linux or macOS, enter the following commands to compile and run the file:
cairo-run --path main.cairo

The string Hello, world! should print to the terminal, with some other information that we don't really need to care for now.

Congratulations! You’ve officially written a Cairo program. Welcome to the world of Cairo!

Hello, Scarb!

Install Scarb

Follow the instructions in the Scarb documentation Current Scarb version is 0.1.0-rc.2, comes with Cairo v0.1.0-alpha.6

Verify installation by running

scarb --version

It should output something like

scarb 0.1.0-rc.2 (8e8d51d80 2023-03-20)
cairo: 1.0.0-alpha.6

Creating a Project with Scarb

Let’s create a new project using Scarb and look at how it differs from our original “Hello, world!” project. Navigate back to your projects directory (or wherever you decided to store your code). Then, on any operating system, run the following:

scarb init hello-scarb

As the result of running scarb new, Scarb has created two files:

  • Scarb.toml - a configuration file for Scarb
  • src/lib.cairo

Common Programming Concepts

Variables and Mutability

By default, variables are immutable. This is one of many nudges Cairo gives you to write your code in a way that takes advantage of the safety and easy concurrency that Cairo offers. However, you still have the option to make your variables mutable. Let’s explore how and why Cairo encourages you to favor immutability and why sometimes you might want to opt out.

When a variable is immutable, once a value is bound to a name, you can’t change that value. To illustrate this, generate a new project called variables in your projects directory by using scarb new variables

Then, in your new variables directory, create src/main.rs and fill with the following code, which won’t compile:

fn main() {
    let x = 5;
    debug::print_felt252(x);
    x = 6;
    debug::print_felt252(x);
}

Save and run the program using cairo-run --path src/main.cairo. You should receive an error message regarding an immutability error, as shown in this output:

error: Cannot assign to an immutable variable.
 --> main.cairo:4:5
    x = 6;
    ^***^

Error: failed to compile: src/main.cairo
